---

Claude Code には CLAUDE.md というルールファイルがあります。プロジェクトのルートディレクトリに `CLAUDE.md` というファイルを作ると、Claude Code が起動時に自動で読み込みます。

ここに書いた指示に従って Claude が動くので、プロジェクトごとの決まりごとや「やってほしくないこと」を書いておくと、毎回説明しなくても意図通りに動いてくれます。

## CLAUDE.md がないとどうなるか

CLAUDE.md がなくても Claude Code は動きます。ただし、こうなります。

- 毎回「このプロジェクトでは TypeScript を使っています」と説明する
- 「そのファイルは消さないで」と言い忘れて消される
- 「デプロイ前に確認して」と言い忘れて本番に反映される
- 前のセッションで伝えたことを次のセッションで覚えていない

CLAUDE.md は「毎回言うのが面倒なこと」を書いておく場所です。

## この本で扱うこと

公式ドキュメントには「CLAUDE.md にプロジェクトの規約を書きましょう」程度のことしか書いてありません。

実際に1ヶ月使うと、こういう疑問が出てきます。

- **何を書くべきか**: 全部書くと読まれない。何を選ぶか
- **どう書くべきか**: 「〜しないこと」と書くのと理由付きで書くのでは効果が違う
- **いつ消すべきか**: ルールは増える一方。消す基準がないと肥大化する
- **どう分けるべきか**: グローバルとプロジェクト固有をどう切り分けるか
- **どう育てるべきか**: 最初に完成させるものではなく、事故のたびに育つもの

この本は、これらの疑問に対して「1ヶ月運用した実物」をもとに答えます。

## 想定読者

- Claude Code を使い始めたが、CLAUDE.md に何を書けばいいか分からない人
- CLAUDE.md を書いたが、効いているのか分からない人
- 他の人の CLAUDE.md を見てみたい人
- AI コーディングツールのカスタマイズに興味がある人

## この本の特徴

- **実物ベース**: 筆者が1ヶ月使った CLAUDE.md の全文（匿名化済み）を掲載
- **判断理由付き**: 「残した」「消した」「変えた」の各ルールに、なぜその判断をしたかを説明
- **テンプレート付き**: すぐに使えるスターターテンプレートをダウンロード可能
- **階層設計**: グローバル ↔ プロジェクト ↔ rules/ ↔ memory の4層構造を体系的に解説
