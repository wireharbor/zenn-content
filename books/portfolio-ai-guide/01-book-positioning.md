note 連載「自分のポートフォリオをAIに相談する」全6記事で「動くまで」の最短ルートを書きました。本書はその続きで、動いた後に出てくるエラー・設計の迷い・壊れる場所・守るべき境界を扱います。

## 本書で積む層

連載の6記事を読んで通した読者が、次に踏む壁を体系化します。

- **エラー図鑑**: インストール・実行時・デプロイ・接続で詰まる場所を、症状→原因→対処のカタログに
- **セキュリティ設計**: 個人の資産データを公開URLで返す是非、認証方式の選び方
- **MCP / FastAPI の内部**: tool annotation、transport、token limit、fastmcp を使うか手書きするか
- **プラットフォーム比較**: Render / Railway / fly.io / Vercel の実運用差と移行手順
- **データ設計**: 毎日更新の仕組み、履歴の持ち方、「今日の変化」に答えるデータ構造
- **実用ユースケース**: 維持率監視、ストレステスト、ニュース連動、リバランス提案

連載が「動かす」、本書が「動き続ける」を扱います。

## 対象読者

3つの層を想定しています。

- **連載を完走した人**: Step 4a/4b まで通したが、毎日運用するうちにエラーや設計の疑問が出てきた
- **連載未読だが同等の経験がある人**: FastAPI + クラウドデプロイ + OpenAPI / MCP の接続を一度は自作したことがある
- **これから始める人**: 第 2 章に連載6記事の要約を置いたので、そこから必要なものを揃えてから戻ってくる

連載と同じく、Python を自分で書く必要はありません。本書のコード例もエージェントに依頼しながら読み進める前提で書いています。

## 前提として持っておくとよいもの

- Claude Code Desktop または Codex のデスクトップアプリが手元で動く
- Google Sheets かローカル CSV に自分の資産データがある
- クラウドサービス（Render など）にデプロイしたAPIのURLを1つ持っている、または第 2 章の要約から連載 Step 3 に戻って作れる
- ChatGPT Plus または Claude Pro 以上のサブスクリプション

これらが揃っていない場合、まず第 2 章（最短ルートのおさらい）を読んでから連載の該当 Step に戻ってください。

## 本書の読み方

章は運用で踏む順に並んでいます。

- **第 I 部（01-02）**: 位置づけと最短ルート要約
- **第 II 部（03-06）**: エラー図鑑
- **第 III 部（07-09）**: セキュリティ設計
- **第 IV 部（10-11）**: MCP と FastAPI の内部
- **第 V 部（12-13）**: プラットフォーム比較と移行
- **第 VI 部（14-16）**: データ設計
- **第 VII 部（17-20）**: 実運用ユースケース
- **第 VIII 部（21-22）**: 振り返りと次の実験候補

章の前後依存は薄めです。自分がいま詰まっている場所（例: Renderがスリープしている、毎日データが古い）から逆引きで読む使い方を想定しています。

## 本書が扱わないこと

- **投資戦略そのもの**: 銘柄選定・構成比・信用取引の具体的な使い方は連載・本書ともに扱いません
- **Python / FastAPI / OpenAPI の入門**: 公式ドキュメントまたは連載 Step 2-4 で充分です
- **GPT Builder / Claude.ai Connector の画面操作**: 連載 Step 4a/4b で画面ごとに解説しています

本書は連載の続きではなく、連載の隣に置く運用ガイドです。連載で糸を通した読者が毎日動かし続けるために必要な判断を積み増す本、と思って読んでください。
