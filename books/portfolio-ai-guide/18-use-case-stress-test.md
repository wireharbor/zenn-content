前章の維持率監視でも登場した「ストレスシナリオ」の話を、この章で本格的に掘ります。

ストレステストは、**「相場が X% 動いたら自分のポートフォリオの NAV / 維持率がどう動くか」を事前に計算しておく** ことです。事後に見るのではなく、起きる前の想定値として持っておきます。相場急変日に初めて計算するのでは遅い、という前提で設計します。

## ストレステストの 3 つの目的

個人でストレステストを運用に組み込む理由:

1. **限界を事前に見える化する**: レバレッジをかけていると、何 % の下落で強制決済されるかを数字で持っておく
2. **相場急変時の行動を事前に決める**: 「-10% で何をする」を相場が動く前に紙に書いておくと、動揺して判断を誤る確率が下がる
3. **AI に根拠付きの回答をさせる**: 「今さらに 5% 動いたらどうなる？」を毎回 LLM に計算させず、事前計算された結果から答えさせる

3 つ目は第 16 章（差分の扱い）の「LLM に計算させない」と同じ思想の続きです。

## 感応度係数という概念

ストレステストの核は **感応度係数** です。「あるファクターが 1% 動いたときに自分の NAV が何 % 動くか」を表す数字で、これさえあれば任意のシナリオに対して線形計算できます。

例:

- 世界株の感応度: **-2.5**（世界株が -1% で NAV -2.5%、世界株が +1% で NAV +2.5%）
- 米 10 年金利の感応度: **-1.9 / 100bp**（米 10Y が +10bp で NAV -0.19%）
- ドル円の感応度: **+0.8**（ドル円が +1% で NAV +0.8%）

感応度は自分のポートフォリオ構造から導出します。

### 感応度の算出方法

- **保有銘柄のベータを加重平均**: 世界株指数に対して各銘柄のベータ（過去の連動度）を計算し、構成比で重み付けして足す
- **過去のリターンから回帰**: 過去 1 年の日次データで、自分の NAV と世界株指数の日次リターンを回帰分析して傾きを取る
- **商品の目論見書から推定**: レバレッジファンドなど、運用方針で「グロス 5.5 倍」とある商品はその倍率を使う

厳密な係数が要るわけではなく、**おおよその数字** で充分です。「世界株 -10% で NAV -20-30% のレンジ」が見えればストレステストの目的は達成します。

## シナリオの設計

単一ファクターのシナリオと、複数ファクターが同時に動くシナリオを両方持ちます。

### 単一ファクター

| シナリオ | 内容 | 想定される状況 |
| --- | --- | --- |
| 世界株 -5% | 世界株のみ -5% | 小さな相場調整 |
| 世界株 -10% | 世界株のみ -10% | 中程度のクラッシュ |
| 世界株 -20% | 世界株のみ -20% | 大規模なクラッシュ |
| 米 10Y +50bp | 金利急騰 | インフレ懸念の高まり |
| 米 10Y -50bp | 金利急落 | 景気悪化懸念 |
| ドル円 -10% | 円高進行 | 日本買い / ドル売り |
| ドル円 +10% | 円安進行 | 日本売り / ドル買い |

### 複合シナリオ

```yaml
Scenario: クラッシュ + 円高 + 金利低下
  world_stock: -15%
  usdjpy: -10%
  us_10y: -30bp

Scenario: インフレ + 円安 + 株高
  world_stock: +5%
  usdjpy: +5%
  us_10y: +50bp

Scenario: 日銀政策変更
  usdjpy: -15%
  world_stock: -5%
```

現実の相場は単一ファクターではなく複合で動きます。複合シナリオを 3-5 個持っておくと、「想定外の動き」が減ります。

## 毎日自動で計算しておく

シナリオを定義したら、日次 cron で毎日計算して JSON に保存します。

```json
{
  "as_of": "2026-04-18",
  "current": {
    "nav_million_jpy": null,
    "maintenance_rate": 185
  },
  "scenarios": [
    {
      "name": "world_stock_-5",
      "nav_change_pct": -12.5,
      "projected_maintenance_rate": 175
    },
    {
      "name": "world_stock_-10",
      "nav_change_pct": -25.0,
      "projected_maintenance_rate": 160
    },
    {
      "name": "world_stock_-20",
      "nav_change_pct": -50.0,
      "projected_maintenance_rate": 115,
      "warning": "forced_liquidation"
    },
    {
      "name": "crash_plus_yen_high",
      "nav_change_pct": -38.0,
      "projected_maintenance_rate": 128,
      "warning": "forced_liquidation"
    }
  ]
}
```

`warning` フィールドで強制決済ラインを下回るシナリオを明示しておくと、AI 側で「このシナリオでは強制決済」と明言できます。

## AI に聞くパターン

事前計算されたストレスシナリオ JSON があると、AI との対話がこうなります。

### ストレスシナリオの直接問い合わせ

```
世界株が 10% 下がったら、私の維持率はどうなる？
```

AI は `scenarios` 配列から `world_stock_-10` を探して「維持率が 160% に低下します。警戒ラインには到達しますが、強制決済ラインの 130% には届きません」のように答えます。

### 相場ニュースとの紐付け

```
今日世界株が 3% 下がったというニュースを見たけど、
私の口座への影響は？
```

AI は 3% を近いシナリオに丸めて（あるいは線形補間して）「世界株 -3% の場合、NAV -7.5%、維持率は約 177% に低下しています」と答えます。線形補間は LLM でなくサーバー側に計算させたい場合、`get_stress_test(shock_pct)` のような tool を足す手もあります。

### 最悪ケースの確認

```
私のポートフォリオで、強制決済までの距離が一番短いシナリオは？
```

AI が全シナリオをスキャンして、`warning: forced_liquidation` のものを抜き出したり、維持率が最も低くなるものを示したりします。事前計算済みなので精度がぶれません。

### 「そろそろ軽くした方がいい？」のような相談

```
今の状況で、ポジションを軽くする判断はどう考えるべき？
```

AI は現在の維持率・最悪シナリオの結果・過去の危険ラインとの距離を総合して答えます。ここは **AI の判断を鵜呑みにしない** 前提で、会話のたたき台として使います。

## シナリオを自分のポートフォリオに合わせる

最初は汎用シナリオ（世界株 -5 / -10 / -20）で充分です。運用が続いてきたら、自分のポートフォリオ特有の弱点を突くシナリオを足します。

- **為替リスクが NAV 比 200% を超えているなら**: ドル円 -15% 単独シナリオ
- **米国株偏重なら**: S&P500 -15% + 円高 10% の組み合わせ
- **金利系商品を持っているなら**: 長短金利スプレッド急縮のシナリオ
- **信用で特定セクターに寄せているなら**: そのセクター固有のクラッシュシナリオ

「自分が怖いシナリオ」を日次で見える化しておくのが目的で、汎用的なテンプレを使い回すより、自分向けにカスタマイズしたほうが効きます。

## 結果をどう振り返るか

ストレステストの予測値と、実際に起きた相場変動の結果を後で突き合わせると、感応度係数の精度がわかります。

- 予測: 世界株 -10% で NAV -25% → 実際: NAV -22%
- 差分: 予測が実測より 3 ポイント大きい（過大推定）

このズレが大きすぎる場合、感応度係数を更新します。これを四半期に 1 度くらいやると、予測と実測のズレが管理下に収まります。過去のドローダウンのトリガーとその深さを記録する JSON を別に持っておくと、実測値の蓄積ができて後で見返しやすくなります。

## 読者が始めるなら

1. 自分のポートフォリオの **世界株に対する感応度係数** だけ先に算出する（1 日で組める）
2. 単一ファクター 3 シナリオ（-5% / -10% / -20%）を日次計算する
3. 維持率との掛け算で警告レベルを付ける
4. 1 ヶ月運用して「予測と実測のズレ」を見る
5. 複合シナリオと他ファクターを段階的に足す

最初から完璧なモデルを作ろうとすると挫折します。世界株だけで 1 ヶ月回して、その後に 1 つずつ足していくのが続く順序です。

次の第 19 章は、このストレステストを **実際の相場ニュースと紐づけて** 運用する話に進みます。「世界株 -5% というニュースを見た」「なぜ下がった」を AI に聞いたときに、自分のポートフォリオへの影響と組み合わせて答えさせる設計です。
