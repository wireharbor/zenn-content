前章で作ったストレステストを、**実際の相場ニュース** と紐づけて使う話です。

「今日世界株が下がった」というニュースを目にしたとき、問いは 2 つ出ます。

1. 何が原因で下がったか
2. 自分のポートフォリオはどう動いたか（動きそうか）

この 2 つを別々のツールで調べていると時間がかかります。AI アプリにまとめて聞いて、両方を 1 つの回答で返してもらうのが本章のユースケースです。

## 汎用ニュース RSS の罠

最初に考えがちなのは「日経や Bloomberg の RSS を取り込んで、毎朝要約させる」設計です。これは **本書の用途では効かない** ことが多いです。

- RSS の大半は自分の保有銘柄に直接関係ない
- 「注目のトレンド」「話題の銘柄」は保有していないものが大半
- AI に読ませるとトークンを大量に消費してタイムアウト

RSS 経由の一般ニュースを AI に読ませる設計は、個人投資家の運用には過剰で、情報量のわりに意思決定に効きません。

## ドライバー起点のニュース連動

代わりに有効なのは **「自分のポートフォリオで何が効いたか」から入って、その理由をニュースで補う** アプローチです。

流れ:

1. 毎日、自分の NAV が動いた主要ドライバー（世界株、米 10Y 金利、ドル円など）を事前計算する（第 16 章）
2. AI に「今日の変化を教えて」と聞く
3. AI はドライバーの数字を返すと同時に、**AI 内蔵の web search で市場ニュースを検索** してドライバーの背景を補足する

この順番だと、AI が読むニュースが「自分に関係するものに絞られる」ため、回答が濃くなります。

例: AI の回答

```
今日の NAV は +0.42% 動きました。主な内訳:

・世界株 -0.4% → NAV +1.0%
  背景: NVDA の好決算で半導体株が上昇。欧州株も堅調。
  
・米 10Y +3bp → NAV -0.58%
  背景: FOMC 議事要旨でタカ派寄りの発言、インフレ再加速警戒

・ドル円 +0.5% → NAV +0.12%
  背景: 日銀政策会合での慎重発言

ネット +0.42%、維持率 185%（前日比変わらず）
```

ドライバーの数字は事前計算されたものを返し、**「なぜ動いたか」だけ AI に取ってこさせる** のが肝です。

## AI の web search を活用する設計

ChatGPT / Claude.ai には web search 機能があり、tool 呼び出しの一部として内部でニュース検索できます。

### MCP tool で「ニュース補足を求める」signal を出す

筆者の「ニュースドライバーを返す tool」は、ドライバーの数字だけ返し、説明文には

```
driver_background: "前日比でこう動いた。直近のニュースで背景を検索して補足してほしい"
```

のような文字列を入れています。これを受け取った Claude / ChatGPT は、自発的に web search をしてドライバーごとの背景を引いてきます。

### web search の指示は description に埋める

tool の description に「この tool は市場ドライバーの数字だけ返します。背景は呼び出し元で web で補足してください」と明示しておくと、AI は tool の結果と web search を組み合わせる振る舞いを取りやすくなります。

### 結果の「根拠 URL」は出させない

ニュースの根拠 URL を AI に引用させるかどうかは好みですが、個人利用で 1 日数回聞くだけなら不要なケースが多いです。「引用リンク付きで返して」と Instructions に入れると、URL の長文が AI の回答を太らせ、読みづらくなります。

## 実運用フロー: 朝の 3 分

筆者が朝やっているフロー:

1. スマートフォンで Claude.ai（または ChatGPT）を開く
2. 「今の状況と、今日動いたドライバーの背景を教えて」と送る
3. 15 秒ほどで、上のような形式の回答が返る
4. 気になるドライバーがあれば深掘りを追加で聞く（「FOMC 議事要旨って具体的には何が？」）
5. 危険ライン / 警戒ラインに近づいていれば、ダッシュボードを開いて深く見る

所要時間は 3 分程度。これを毎朝することで、自分のポートフォリオの「今どこにいるか」と「市場で何があったか」が同じ粒度で頭に入ります。

## 個別銘柄レベルのニュース連動を避ける

「A 銘柄について今日のニュースは？」を AI にさせる設計は、個人運用では避けるほうが無難です。

- 個別銘柄名を AI に送ると、第 9 章の個人情報開示方針と抵触する
- 個別銘柄のニュースは RSS でも AI の web search でも拾えるが、判断に直接効くケースが少ない
- 銘柄レベルのニュース連動は時間が吸われる

集計ドライバー（世界株 / 米 10Y / ドル円）の単位で済ませるのが、個人運用のちょうどよいバランスです。

## 注意: ニュース解釈は AI の「現在の理解」

AI が web search で持ってくる解釈には、以下のバイアスが含まれ得ます。

- 過去 24-48 時間で最も拾いやすい記事に引っ張られる
- 英語ニュースと日本語ニュースで解釈が違う場合、どちらかに寄る
- センセーショナルな見出しを拾いやすい

「AI がこう言ったから正しい」ではなく、**AI はニュースの要約係** として扱います。重要な判断をするときは、自分でソースを見に行きます。

## 具体的な問いの例

朝・昼・夜で自分が投げている代表的な問いをそのまま紹介します。

```
今の状況と、今日動いたドライバーの背景を教えて
```

```
世界株が下がったと聞いたけど、なぜ？
私のポートフォリオへの影響は？
```

```
FOMC の結果で米 10Y が跳ねたらしいけど、私のポートフォリオはどう動く？
```

```
日銀の政策変更で円高が進んだ場合、
私のポートフォリオで一番影響を受けるのはどのドライバー？
```

```
今月のパフォーマンスを振り返って、
大きく効いたドライバーを 3 つ挙げて
```

どれも「自分のドライバー × 市場ニュース」の組み合わせで、AI が事前計算済みの数字と web search を噛み合わせて答えます。

## 読者が組むなら

1. 自分のポートフォリオの **主要ドライバー（3-5 個）** を決める
2. それぞれの感応度係数を第 18 章の方法で算出
3. 日次 cron で「今日の動きのドライバー別分解」を事前計算
4. MCP tool にドライバーを返すエンドポイントを足す（description に「背景はニュース検索で補足」と書く）
5. 1 週間スマホから使ってみて、ドライバー粒度が合っているか確認
6. 合わなければドライバーを差し替えて繰り返す

ドライバーを自分のポートフォリオに合うよう調整するのが、このユースケースの精度のほとんどを決めます。汎用指数（世界株、日経、ドル円）で始めて、合わなければ「新興国株」「米テクノロジー株」「エネルギーセクター」など細分化するのが流れです。

次の第 20 章は第 VII 部（ユースケース）の最後で、**定期リバランス** の提案を AI にどう任せるかの話です。提案までは AI、実行は自分、という役割分担の設計を書きます。
